    General information

    Introduction

    Welcome to the 8keys API Docs! You can use our API to get information on your balance, products and orders.

    In the dark area to the right you will find a sample implementation with PHP and Guzzle.

    Some general information about the API:

    • Content-Type: application/json or application/x-www-form-urlencoded (we strongly recommend using json!)
    • Response Format: JSON, make sure to add Accept: application/json to all your API requests to the server!
    • Money Format: Cents as an integer, f.e. 12972 for 129.72 EUR.
    • DateTime Format: YYYY-MM-dd HH:mm:ss, f.e. 2020-04-03 18:04:00

    Environments:

    Name Description
    Development
    Dev    		 Environment to develop / test with virtual balance which can be topped up with one click.
    Production
    Prod    		 Live environment for productive use with real balance which can be topped up via bank transfer / PayPal.

    Authentication

    Generate API keys

    To (re-)generate your API keys, go to My Stores and click on in the Actions column. Confirm the procedure and be sure to note your api keys down in a safe place!

    Authorization

    Client implementation (used in the following code examples)

    $api_client = new \GuzzleHttp\Client([
    'base_uri' => 'https://api.8keys.de/d/v1/',
    'timeout'  => 60.0,
    'http_errors' => false,
    'headers'  => [
        'Authorization' => 'Bearer EXAMPLE-API-KEY',
        'Accept' => 'application/json',
        'Content-Type' => 'application/json'
    ],
]);

    The API key must be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer EXAMPLE-API-KEY

    You must replace EXAMPLE-API-KEY with your personal API key.

    Rate Limit

    To protect our servers from too many requests, we have a rate limit of 60 requests per minute. The rate limit applies to each environment separately.

    You can check the current rate limit status via the response headers:

    x-ratelimit-limit: 60 and x-ratelimit-remaining: 34

    If you reach the rate limit, the following requests will be blocked with 429 Too Many Requests and you can use the following response header to check when the rate limit is reset (UNIX Timestamp):

    x-ratelimit-reset: 1591463645

    Product-Key Types

    We use MIME Types to describe the format of our product keys.

    The most used types are the following:

    Type Notes Example
    text/plain Serial key as a text ABCD-1234-7890-4217
    image/jpeg
    image/png
    image/gif    		 Serial key as an image (base64 encoded)

    Error Codes

    The 8keys API uses the following error codes:

    Error Code Description
    302 Found You have probably forgotten the Accept: application/json header
    401 Unauthorized The store has not yet been activated or the API token is invalid.
    404 Not Found Route or specified product/order not found.
    405 Method Not Allowed You tried to access a resource with an invalid method.
    422 Unprocessable Entity Your request query / body data is invalid (see response body for more details).
    429 Too Many Requests You're sending too many requests! Slow down!
    500 Internal Server Error We had a problem with our server. Try again later.

    Products

    Get All Products

    $api_client->get('products', [
  'query' => [
    'page' => 4,
    'limit' => 50,
    'name' => 'Win',
    'productIds' => [ 1, 4, 7, 8 ],
    'updatedSince' => '2000-04-03 00:00:00'
  ]
]);

    The above command returns JSON structured like this:

    // -> Status Code: 200 OK
{
  "total_count": 350,
  "result": [
    {
      "id": 1,
      "name": "Anno 2070 Uplay CD Key",
      "description": "Lorem ipsum dolor sit amet, consetetur sadipscing elitr, sed diam nonumy.",
      "cover": "https:\/\/api.8keys.de\/example-cover.jpeg",
      "platform": "Uplay",
      "language": "German, English, French, Spanish, Czech, Polish, Russian, Italian",
      "region": "Region free",
      "quantity": 4129,
      "price_cents": 615,
      "updated_at": "2020-04-24 13:54:18"
    },
    [...]
  ]
}

    This endpoint retrieves all products.

    HTTP Request

    GET /products

    Query Parameters

    Parameter Type Default Required Description
    page int 1 false Current page (>= 1)
    limit int 10 false Limit product count (between 1 and 100)
    name string NULL false Filter by product name (*wildcard*)
    productIds array of int NULL false Filter by product ids (max. 100 ids)
    updatedSince datetime NULL false Get products updated since datetime

    Orders

    Get All Orders

    $api_client->get('orders', [
  'query' => [
    'page' => 4,
    'limit' => 50,
    'createdAfter' => '2000-04-03 00:00:00'
  ]
]);

    The above command returns JSON structured like this:

    // -> Status Code: 200 OK
{
  "total_count": 12,
  "result": [
    {
      "id": 1,
      "transaction_id": 1,
      "status": "done",
      "created_at": "2020-04-07 20:12:36"
    },
    [...]
  ]
}

    This endpoint retrieves all orders (without product keys).

    HTTP Request

    GET /orders

    Query Parameters

    Parameter Type Default Required Description
    page int 1 false Current page (>= 1)
    limit int 10 false Limit order count (between 1 and 100)
    createdAfter datetime NULL false Get orders created after datetime

    Get Specific Order

    $api_client->get('orders/1');

    The above command returns JSON structured like this:

    // -> Status Code: 200 OK
{
  "id": 1,
  "transaction_id": 1,
  "status": "done",
  "created_at": "2020-04-07 20:12:36",
  "product_keys": [
    {
      "product_id": 1,
      "type": "text/plain",
      "code": "0000-0000-0000-0000"
    },
    [...]
  ],
}

    This endpoint retrieves a specific order.

    HTTP Request

    GET /orders/{ID}

    URL Parameters

    Parameter Description
    ID The ID of the order to retrieve
    The product keys will only be included if the order status is done

    Place Order

    $api_client->post('orders', [
  'json' => [
    'products' => [
      [
        'product_id' => 1,
        'quantity' => 4,
        'price_cents' => 615
      ],
      [...]
    ]
  ]
]);

    The above command returns JSON structured like this:

    // -> Status Code: 201 Created
{
  "order_id": 14
}

    This endpoint places a new order. Please do not exceed a maximum of 50 units per order.

    HTTP Request

    POST /orders

    Body Parameters

    Parameter Type Default Required Description
    products array of products - true Requested products
    products.*.product_id int - true Product ID
    products.*.quantity int - true Desired Quantity (max. 50)
    products.*.price_cents int - true Price in cents (for data reconciliation)
